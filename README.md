# Projeto-Final-IHC

---

## 1. Objetivo do Projeto

Este projeto tem como objetivo o desenvolvimento de uma interface usuário, considerando aspectos de usabilidade, acessibilidade e experiência do usuário (UX). A proposta é criar uma solução que facilite a interação entre o usuário e o sistema, com foco na necessidade de integração entre os componentes front-end e back-end, além de promover a inclusão e acessibilidade, atendendo às demandas dos usuários com diferentes perfis.

## 3. Descrição do Projeto

O projeto de IHC é focado na criação de uma interface funcional, com validações de usabilidade e acessibilidade, para um sistema que possibilite o mapeamento e interação entre os usuários e suas necessidades. Durante o desenvolvimento, serão realizadas várias fases de prototipagem e testes de UX, incluindo validações com usuários reais, para garantir que a experiência oferecida seja otimizada.
Principais Etapas:
Pesquisa e Análise de Requisitos: Levantamento de dados sobre as necessidades do usuário e análise de sistemas semelhantes.

Desenvolvimento de Wireframes e Protótipos: Criação de wireframes de baixa e alta fidelidade.

Validação de UX: Avaliação dos protótipos com usuários, com base em técnicas como walkthrough e card sorting.

Implementação Técnica: Desenvolvimento da interface e integração com o back-end.

Testes de Usabilidade e Acessibilidade: Validação da funcionalidade e acessibilidade da interface.


## 4. Contexto Geral

### 4.1 Definição de Usuários e Atividades

Durante o projeto, foram criadas personas que representam o perfil dos usuários. A análise dessas personas ajudou a mapear as atividades essenciais do sistema, como interação com formulários, navegação entre páginas e realização de buscas.

### 4.2 Análise de Usabilidade e Acessibilidade

A análise de usabilidade foi feita com base nas expectativas dos usuários e no feedback recebido durante os testes de validação. Também foi priorizada a acessibilidade, garantindo que a interface seja utilizável por todas as pessoas, incluindo aquelas com necessidades especiais.

## 6. Estrutura do Repositório

A estrutura do repositório segue uma organização modular, com separação de responsabilidades entre as pastas do frontend, backend e documentação:

- **/frontend**: Interface do usuário, componentes, páginas, estilos, serviços e assets.
- **/backend**: API, regras de negócio, integração com banco de dados e testes do servidor.
- **/docs**: Documentação do projeto.
    - **/docs/analise**: Mapa de empatia, requisitos funcionais e não funcionais.
    - **/docs/design**: Protótipos, wireframes, fluxos e guia de estilos.
    - **/docs/relatorios**: Relatórios (parcial/final), atas e apresentações.
- **/database**: Scripts SQL, migrações e seeds para popular o banco.

## 7. Metodologia de Desenvolvimento

Para a construção da interface, adotamos o método ágil, utilizando sprints curtas para entrega de funcionalidades parciais e testes contínuos. Ao longo do desenvolvimento, realizamos ciclos de iteração onde os protótipos foram constantemente avaliados e ajustados conforme o feedback dos usuários.

- **Testes de Usabilidade**
Métodos Aplicados: Testes de usabilidade com usuários reais utilizando cenários de uso para validar a eficácia das funcionalidades.

Feedback Recebido: A interface foi ajustada de acordo com as dificuldades encontradas pelos usuários durante a navegação, como a navegação confusa e dificuldades em localizar funções específicas.


- **Testes de Acessibilidade**
Ferramentas Utilizadas: Ferramentas como WAVE e Axe foram usadas para avaliar a conformidade com as diretrizes WCAG (Web Content Accessibility Guidelines).

Resultados e Ajustes: Acessibilidade foi otimizada, com ajustes como contraste adequado, navegação por teclado e suporte a leitores de tela.
